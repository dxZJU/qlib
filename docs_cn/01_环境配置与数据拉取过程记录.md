## Step 2：创建 Conda 虚拟环境

Qlib 官方建议使用 Python 3.8–3.12，并推荐在 conda 环境中管理依赖，以避免头文件缺失导致的编译失败。

```powershell
conda create -n qlib python=3.9 -y
conda activate qlib
```

**执行结果**：
- 环境名称：`qlib`
- Python 版本：3.9
- 环境路径（示例）：`E:\Anaconda\envs\qlib`

---

## Step 3：安装基础依赖（numpy / cython）

在从源码安装 Qlib 之前，必须先安装 `numpy` 和 `cython`，因为 Qlib 的 C 扩展（`.pyx` 文件）在 build 时需要它们。

```powershell
pip install numpy cython
```

**注意**：如在 Mac M1 上安装，还需要先安装 openmp：
```bash
brew install libomp
```

## Step 4：安装 Qlib

### 方案 A（Windows 推荐）：PyPI 预编译 wheel

Qlib 包含两个 Cython C++ 扩展（`_libs/rolling.pyx` 和 `_libs/expanding.pyx`），Windows 上从源码编译需要 Visual C++ Build Tools 14.0+。如未安装，**直接使用 PyPI 预编译 wheel**：

```powershell
# 第一步：单独安装 greenlet（也需要预编译 wheel）
pip install "greenlet>=2.0.0" --only-binary=:all:

# 第二步：下载 Python 3.9 Windows x64 预编译 wheel
pip download pyqlib --no-deps --python-version 39 --platform win_amd64 --only-binary=:all: -d C:\Temp\pyqlib_wheel

# 第三步：安装（wheel 文件包含预编译的 .pyd C 扩展）
pip install C:\Temp\pyqlib_wheel\pyqlib-0.9.7-cp39-cp39-win_amd64.whl

# 额外安装 LightGBM
pip install lightgbm
```

**执行日志（实录）**：
```
Successfully installed pyqlib-0.9.7 mlflow-3.1.4 lightgbm-4.6.0 ...
```
> ⚠️ **遇到的问题**：先后尝试 `pip install D:\vscode_project\qlib` 和 `pip install pyqlib`，均因 `greenlet` 需要 MSVC 编译失败。最终方案：先预装 `greenlet` 预编译 wheel，再下载对应 Python 版本的 `pyqlib` wheel 离线安装。

### 方案 B（Linux/Mac 或已安装 MSVC）：从源码安装

```bash
pip install numpy cython
pip install D:\vscode_project\qlib    # Windows
# 或 pip install /path/to/qlib       # Linux/Mac
```

若用于开发调试，推荐 editable 模式：
```bash
pip install -e .[dev]
```

## Step 6：下载数据

### 6.1 方案 A（推荐）：社区数据源 chenditc/investment_data

由于 Qlib 官方数据集因数据安全政策暂时下线，README 推荐使用社区维护的 [chenditc/investment_data](https://github.com/chenditc/investment_data/releases) 数据源。

**Linux/Mac**：
```bash
wget https://github.com/chenditc/investment_data/releases/latest/download/qlib_bin.tar.gz
mkdir -p ~/.qlib/qlib_data/cn_data
tar -zxvf qlib_bin.tar.gz -C ~/.qlib/qlib_data/cn_data --strip-components=1
rm -f qlib_bin.tar.gz
```

**Windows PowerShell（实录，使用 BITS 下载 + Python 解压）**：

```powershell
# 步骤 1：创建目标目录
New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\.qlib\qlib_data\cn_data"

# 步骤 2：使用 BITS 异步下载（522.7 MB，约 5-15 分钟，取决于网速）
$url = "https://github.com/chenditc/investment_data/releases/download/2026-04-19/qlib_bin.tar.gz"
$dest = "$env:USERPROFILE\.qlib\qlib_data\qlib_bin.tar.gz"
Start-BitsTransfer -Source $url -Destination $dest -Asynchronous

# 步骤 3：检查下载进度
Get-BitsTransfer | Where-Object {$_.DisplayName -eq 'BITS Transfer'} | 
    Select-Object JobState, @{N='MB';E={[math]::Round($_.BytesTransferred/1MB,1)}}, @{N='TotalMB';E={[math]::Round($_.BytesTotal/1MB,1)}}

# 步骤 4：完成传输（Transferred 状态时执行）
Complete-BitsTransfer -BitsJob (Get-BitsTransfer | Where-Object {$_.DisplayName -eq 'BITS Transfer'})

# 步骤 5：用 Python 解压（避开 Windows 无内置 tar.gz 支持）
python - << 'EOF'
import tarfile, os
src = os.path.expanduser(r'~\.qlib\qlib_data\qlib_bin.tar.gz')
dst = os.path.expanduser(r'~\.qlib\qlib_data\cn_data')
os.makedirs(dst, exist_ok=True)
with tarfile.open(src, 'r:gz') as t:
    for m in t.getmembers():
        parts = m.name.split('/', 1)
        if len(parts) > 1 and parts[1]:
            m.name = parts[1]
            t.extract(m, dst)
print('Done.')
EOF
```

> **实际耗时**：下载约 5 分钟（522.7 MB），解压约 10 分钟（66913 个文件）

**下载完成后验证**：
```powershell
(Get-Item "$env:USERPROFILE\.qlib\qlib_data\qlib_bin.tar.gz").Length / 1MB
# 输出：522.664293289185  ✓
Get-ChildItem "$env:USERPROFILE\.qlib\qlib_data\cn_data" | Select-Object Name
# 输出：calendars  features  instruments  ✓
```

## 附录：常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| `Microsoft Visual C++ 14.0 required` | Windows 无 MSVC 编译器，Cython 扩展编译失败 | 改用 `pip install "greenlet>=2.0.0" --only-binary=:all:` 再用 wheel 安装 pyqlib |
| `conda run` 拒绝多行 `-c` 脚本 | conda 23.x 不支持含换行符的参数 | 将脚本写入 `.py` 文件后执行 |
| `pyarrow` 缺失 | 默认未安装 parquet 引擎 | `pip install pyarrow` |
| `lightgbm` 在 Mac M1 编译失败 | 缺少 openmp | `brew install libomp` 后重装 |
| 官方数据下载链接不可用 | 安全策略暂时下线 | 使用社区数据源（Step 6.1），版本日期见 releases 页面 |
| `qrun` 命令不存在 | qlib 安装不完整 | 重新执行 `pip install C:\path\to\pyqlib.whl` |
| `ImportError: Do not import qlib in repo dir` | 在 `qlib/` 父目录下运行 | 切换到 `examples/` 目录后再运行 `qrun` |
| BITS 下载显示 "Connecting" 状态 | 正在建立连接，正常等待即可 | 几分钟后用 `Get-BitsTransfer` 检查 JobState |
