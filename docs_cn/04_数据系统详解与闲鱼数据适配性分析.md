## Part 1：Qlib 数据系统详解

### 1.1 数据格式：`.bin` 文件（Qlib 格式）

Qlib 使用自研的二进制格式（`.bin`）存储时间序列数据。该格式具备以下特点：

- **高性能**：专为科学计算设计，避免通用数据库的格式转换开销。实测在 1 CPU 下处理 800 支股票 14 年数据仅需 **7.4 秒**（MySQL 需 184 秒）。
- **时间对齐**：所有特征序列均对齐到交易日历（`calendars/day.txt`），缺失日期自动填 NaN。
- **复权处理**：价格数据为**前复权价格**，每支股票的第一个交易日价格归一化为 1。
  - 公式：`原始价格 = 复权价格 / factor`
  - 其中 `factor` 字段即复权因子（前复权因子）。

### 1.2 数据目录结构


### 1.4 数据字段（Features）

Qlib 官方数据集提供的基础字段：

| 字段名 | 含义 | 备注 |
|--------|------|------|
| `open` | 开盘价（复权） | 前复权 |
| `close` | 收盘价（复权） | 前复权 |
| `high` | 最高价（复权） | 前复权 |
| `low` | 最低价（复权） | 前复权 |
| `volume` | 成交量（复权） | 前复权 |
| `factor` | 复权因子 | `factor = 复权价 / 原始价`，注意 Qlib 归一化第一个交易日价格为 1 |

> **注意**：Qlib 官方数据中**不包含** `vwap`（成交量加权均价）、`amount`（成交额）、`pre_close`（昨收）等字段，但这些字段可以在自己导入数据时添加。

> **暂停停牌处理**：若股票当日停牌，则 `open`, `close`, `high`, `low`, `volume`, `amount`, `factor` 全部设为 NaN。

### 1.5 数据频率

| 频率 | 下载参数 | 存储路径 | 说明 |
|------|---------|---------|------|
| 日线（1d） | `--interval 1d`（默认） | `~/.qlib/qlib_data/cn_data/` | 标准 benchmark 使用 |
| 1 分钟线（1min） | `--interval 1min` | `~/.qlib/qlib_data/cn_data_1min/` | 高频示例使用 |

### 1.6 预定义数据集（Alpha 因子集）

Qlib 提供两个开箱即用的因子集，均基于日线 OHLCV 数据通过表达式引擎计算：

#### Alpha360

- **因子数量**：360 个
- **构造方式**：过去 60 天的 `close/open/high/low/vwap/volume`，每个字段各 60 天滚动，所有价格均以当日 `$close` 归一化
- **特点**：包含较多原始价格信息，适合深度学习模型

示例因子（部分）：
```
CLOSE0 = $close/$close  (= 1.0)
CLOSE1 = Ref($close, 1)/$close  (昨日收盘/今日收盘)
...
CLOSE59 = Ref($close, 59)/$close
OPEN0 = $open/$close
VOLUME0 = $volume/($volume+1e-12)
```

#### Alpha158

- **因子数量**：158 个
- **构造方式**：基于 K 线形态、滚动统计、价格动量等，包括但不限于：
  - K 线特征（KBAR）：如日内振幅、影线比例等
  - 价格特征（PRICE）：`OPEN0`=`$open/$close`，`HIGH0`=`$high/$close` 等
  - 滚动特征（ROLLING）：`ROC5`=5 日收益率、`MA5`=5 日均价、`STD5`=5 日波动率等，窗口为 5/10/20/30/60
- **标签（Label）**：`Ref($close, -2) / Ref($close, -1) - 1`，即 T+2 相对 T+1 的收益率（考虑 A 股 T+1 买入限制）
- **特点**：因子含义明确，适合 GBDT 类模型

### 1.7 数据健康性检查机制

Qlib 提供 `scripts/check_data_health.py` 脚本，检查以下项目：

1. DataFrame 中是否存在缺失数据（超过阈值则报警）
2. OHLCV 列是否存在超大跳变（可能的数据错误）
3. 必需列（OHLCV）是否完整
4. `factor` 列是否存在

## Part 3：适配性评估

### 3.1 Qlib 自定义数据导入要求

根据 Qlib 官方文档（[Converting CSV/Parquet Format into Qlib Format](https://qlib.readthedocs.io/en/latest/component/data.html#converting-csv-format-into-qlib-format)），自定义数据要求：

| 要求 | 说明 |
|------|------|
| **文件格式** | CSV 或 Parquet |
| **文件命名** | 以股票代码命名（如 `SH600000.csv`）**或** 数据中包含 `symbol` 字段 |
| **必须包含** | 日期字段（可指定列名）、OHLCV 字段 |
| **强烈建议** | 包含 `factor`（复权因子）字段 |
| **停牌处理** | 停牌日 OHLCV 需设为 NaN |

转换命令示例：
```bash
python scripts/dump_bin.py dump_all \
    --data_path ~/.qlib/my_data \
    --qlib_dir ~/.qlib/qlib_data/ \
    --include_fields open,close,high,low,volume,factor \
    --file_suffix .parquet
```

### 3.2 闲鱼数据 vs Qlib 要求：逐项对比

| Qlib 要求 | 闲鱼数据状态 | 适配程度 |
|-----------|------------|---------|
| 文件格式为 CSV/Parquet | ✅ 数据为 Parquet 格式 | **完全符合** |
| 包含日期字段 | ✅ 有 `date` 和 `trade_time` 字段 | **完全符合** |
| 包含 `open, close, high, low` | ✅ 都有 | **完全符合** |
| 包含 `volume` | ⚠️ 字段名为 `vol`（非 `volume`） | 需通过 `--symbol_field_name` 或重命名适配 |
| 包含 `factor`（复权因子） | ❌ **不包含** factor 字段 | **不符合** |
| 文件按股票代码命名 | ❌ 文件按日期命名（`20241212.parquet`），一个文件含所有股票 | 需重组：按股票切分文件，或用 `--symbol_field_name code` |
| 股票代码格式 | ⚠️ 使用 `000001.SZ` 格式（Tushare 风格），Qlib 使用 `sz000001` 格式 | 需要代码格式转换 |
| 停牌日设为 NaN | ❓ 数据来源为商业数据，停牌处理方式不明 | 需验证 |
| 数据为日线 | ❌ 是**分钟线**，Qlib 默认 benchmark 使用日线 | 适配分钟线需额外工作（`--interval 1min`） |

### 3.3 关键差距分析

#### 3.3.1 缺少 `factor`（复权因子）— 最大挑战

Qlib 的价格复权机制依赖 `factor` 字段。没有 `factor` 意味着：
- 无法让 Qlib 的表达式引擎通过 `$close / $factor` 还原原始价格
- Qlib 会在数据健康性检查中报警

**解决方案**：
- 方案 A：如果闲鱼数据本身已是**未复权的原始价格**，可将 `factor` 设为全 1.0，并标注数据为未复权
- 方案 B：如果数据已复权，需要找到对应的复权因子（可考虑从 baostock 或 tushare 获取历史复权因子对齐）
- 方案 C：对于纯分钟线研究，若不需要跨日复权，可将每日价格序列视为原始数据，`factor` 设 1.0

#### 3.3.2 数据组织方式不同

闲鱼数据是"**宽表按日期**"组织（一个文件含所有股票当日所有分钟），Qlib 要求"**按股票**"组织（每股票一个文件或通过 `symbol` 字段区分）。

**转换脚本示例**：
```python
import zipfile, pandas as pd
from pathlib import Path

output_dir = Path("~/.qlib/my_data").expanduser()
output_dir.mkdir(parents=True, exist_ok=True)

# 遍历所有年份 zip
for year_zip in Path("A股分钟数据/1m_price_zip").glob("*.zip"):
    with zipfile.ZipFile(year_zip) as z:
        for entry in z.namelist():
            with z.open(entry) as f:
                df = pd.read_parquet(f)
            # 重命名字段
            df = df.rename(columns={"vol": "volume"})
            # 添加 factor 字段（未复权情况下设为 1）
            df["factor"] = 1.0
            # 按股票分组，每股票追加写入
            for code, group in df.groupby("code"):
                # 转换代码格式：000001.SZ → sz000001
                qlib_code = code.split(".")[1].lower() + code.split(".")[0]
                out_file = output_dir / f"{qlib_code}.parquet"
                group.to_parquet(out_file, append=True if out_file.exists() else False)
```

#### 3.3.3 分钟线 vs 日线

Qlib 的主流 benchmark（Alpha158/Alpha360、LightGBM、LSTM 等）全部基于**日线**数据。闲鱼数据为分钟线，若要使用 Qlib：

- **选项 A**：将分钟线 OHLCV 重采样为日线（使用第一分钟开盘/最后分钟收盘/最高/最低/累加成交量），再按日线导入 Qlib
- **选项 B**：直接用于 Qlib 的高频数据管道（`highfreq_handler.py`），但配置复杂度更高
- **选项 C**：基于分钟数据自建特征，不依赖 Alpha158/Alpha360，仅使用 Qlib 的回测引擎

### 3.4 与 Yahoo Finance 数据的质量对比

Qlib README 明确警告：
> "Please pay ATTENTION that the data is collected from Yahoo Finance, and the data might not be perfect. We recommend users to prepare their own data if they have a high-quality dataset."

| 比较维度 | Yahoo Finance（Qlib 默认） | 闲鱼商业数据 |
|---------|--------------------------|------------|
| **数据精度** | 已知有错误、延迟和缺失 | 商业来源，质量较高 |
| **复权处理** | 有 factor 字段，前复权 | 需确认是否复权以及复权方式 |
| **数据频率** | 日线为主，1min 可选 | 1min、5min、日线均有 |
| **覆盖时间** | 2007 年至今 | 2010 年至今（部分年份缺失） |
| **覆盖范围** | A 股全市场 + 美股 | A 股全市场 + ETF + 指数 |
| **成本** | 免费 | 需购买 |

闲鱼数据在**数据精度和时效性**上优于 Yahoo Finance，是 Qlib README 所建议"自有高质量数据集"的合适替代。

## Part 4：适配路径建议

### 推荐适配方案（完整步骤）

```
1. 数据解压与重组
   ├── 逐年解压 zip → 读取日期.parquet
   ├── 将宽表（按日期）转为长表（按股票）
   └── 每只股票输出一个 parquet 文件（stock_code.parquet）

2. 字段映射
   ├── vol → volume
   ├── trade_time → 作为日期字段（分钟线用）或从 date 提取
   └── 添加 factor=1.0（或获取真实复权因子后填入）

3. 代码格式转换
   └── 000001.SZ → sz000001（Qlib 命名规范）

4. 使用 dump_bin.py 转换为 .bin 格式
   python scripts/dump_bin.py dump_all \
       --data_path <重组后的数据目录> \
       --qlib_dir ~/.qlib/qlib_data/cn_data_xianyu \
       --include_fields open,close,high,low,volume,factor \
       --file_suffix .parquet \
       --date_field_name trade_time \
       --symbol_field_name code

5. 初始化 Qlib 使用自定义数据
   qlib.init(
       provider_uri="~/.qlib/qlib_data/cn_data_xianyu",
       region=REG_CN
   )
```

### 适配性总结

| 评估结论 | 内容 |
|---------|------|
| **可以适配** | 闲鱼数据的格式、字段内容与 Qlib 要求高度兼容，经过一定转换工作后可以导入 |
| **主要挑战** | ① 缺少复权因子 `factor`；② 数据按日期组织需重排为按股票组织；③ 代码格式需转换 |
| **质量评估** | 作为商业数据，质量显著优于 Qlib 默认的 Yahoo Finance 数据，满足 README 中"自有高质量数据"的建议 |
| **频率适配** | 分钟数据直接用于 Qlib 标准 benchmark 需先重采样为日线，或改用 Qlib 高频数据管道 |
| **工作量估计** | 编写适配脚本约 200–400 行代码，主要工作量在数据重组和验证 |
