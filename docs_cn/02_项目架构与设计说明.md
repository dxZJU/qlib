## 1. 项目概述

**Qlib** 是微软亚洲研究院开源的、以 AI 为中心的量化投资平台。其核心目标是：

> 利用 AI 技术赋能量化投资研究，覆盖从 idea 探索到生产落地的完整链路。

Qlib 支持三类学习范式：
- **监督学习（Supervised Learning）**：预测股票收益、挖掘 alpha 因子
- **市场动态建模（Market Dynamics Modeling）**：处理分布漂移、在线自适应
- **强化学习（Reinforcement Learning）**：建模连续交易决策、订单执行优化

## 2. 整体架构

Qlib 采用**松耦合分层架构**，各层可独立使用。从底层到高层依次为：

```
┌─────────────────────────────────────────────────────────┐
│               在线服务层 Online Serving                  │
│     模型上线 / 实时预测 / 在线策略更新                    │
├─────────────────────────────────────────────────────────┤
│               分析层 Analysis & Report                   │
│     IC分析 / 回测报告 / 组合归因 / 可视化                 │
├─────────────────────────────────────────────────────────┤
│               工作流层 Workflow                          │
│   任务编排(qrun) / 实验记录(MLflow) / 模型训练/预测/回测  │
├───────────────────────────┬─────────────────────────────┤
│    预测模型 Forecast Model │   交易策略 Trading Strategy  │
│  监督学习模型/RL Agent      │  TopkDropout/RL策略         │
├───────────────────────────┴─────────────────────────────┤
│               学习框架层 Learning Framework              │
│        监督学习框架 / 强化学习框架 / 元学习框架            │
├─────────────────────────────────────────────────────────┤
│               数据层 Data Layer                         │
│   数据加载 / 特征工程 / 数据集构建 / 缓存机制             │
├─────────────────────────────────────────────────────────┤
│               基础设施层 Infrastructure                  │
│        数据存储（.bin） / 配置系统 / 日志 / 工具函数       │
└─────────────────────────────────────────────────────────┘
```

## 4. 核心源码包：`qlib/`

### 4.1 顶层文件

| 文件 | 功能 |
|------|------|
| `__init__.py` | 模块入口，定义 `qlib.init()` 全局初始化函数，负责加载配置、挂载数据提供者 |
| `config.py` | 全局配置类 `QlibConfig`，管理数据路径、缓存策略、在线/离线模式等参数 |
| `constant.py` | 全局常量（`REG_CN`, `REG_US` 等区域代码，`Freq` 频率定义等） |
| `log.py` | 统一日志模块，封装 Python logging |
| `typehint.py` | 类型注解定义 |

### 4.2 `qlib/data/` — 数据层

数据层是 Qlib 的核心基础设施，负责：原始数据存储、特征表达式计算、数据集构建、缓存加速。

```
qlib/data/
├── __init__.py        # 导出 D（Data Provider 全局入口）
├── data.py            # DataProvider, LocalDataProvider, ClientDataProvider
├── base.py            # Feature, ExpressionOps 基类
├── ops.py             # 全部表达式算子（Ref, Mean, Std, Rank, etc.）
├── filter.py          # NameDFilter, ExpressionDFilter（股票池过滤）
├── cache.py           # MemCache, DiskExpressionCache, DiskDatasetCache
├── client.py          # 在线模式客户端（连接 Qlib-Server）
├── inst_processor.py  # 个股级数据后处理
├── pit.py             # Point-In-Time 数据库（防止数据穿越）
├── storage/           # 底层 .bin 文件读写
│   ├── storage.py     # CalendarStorage, InstrumentStorage, FeatureStorage
│   └── ...
├── dataset/           # 数据集构建
│   ├── __init__.py    # DatasetH（带 Handler 的 Dataset）
│   ├── handler.py     # DataHandlerLP（可学习处理器的 Handler）
│   ├── loader.py      # QlibDataLoader, StaticDataLoader
│   ├── processor.py   # 标准化处理器（ZscoreNorm, DropnaLabel 等）
│   └── weight.py      # 样本权重（Reweighter）
└── _libs/             # Cython 编译的高性能数据读取 C 扩展
```

**数据流向**：

```
.bin 文件（磁盘）
  → FeatureStorage（storage/）读取原始数据
  → ops.py 中的表达式引擎计算特征（如 Mean($close, 5)）
  → ExpressionCache（cache.py）缓存计算结果
  → DataHandlerLP（dataset/handler.py）做归一化等处理
  → DatasetH（dataset/__init__.py）按 train/valid/test 切分
  → 模型消费
```

**关键概念**：

- **`D`（Data Provider）**：全局数据访问入口，`from qlib.data import D`。支持 `D.features()`, `D.calendar()`, `D.instruments()` 等接口。
- **表达式引擎**：通过字符串表达式（如 `"Ref($close, 1) / $close - 1"`）声明特征，引擎自动计算并缓存。支持的算子见 `qlib/data/ops.py`（约 50+ 个）。
- **Alpha158/Alpha360**：两个开箱即用的特征集，定义在 `qlib/contrib/data/loader.py`。Alpha158 有 158 个因子，Alpha360 有 360 个因子。
- **DataHandlerLP**：`LP` 代表 Learnable Processors，支持 fit/transform 流程，可在训练集上 fit 参数（如均值/方差），在推理集上 transform。
- **Point-In-Time（PIT）**：`pit.py` 实现了防数据穿越的数据库，适合财务指标（PE、EPS 等）的使用。

### 4.3 `qlib/model/` — 模型基类

```
qlib/model/
├── base.py       # Model（预测接口）, ModelFT（支持 fine-tune）基类
├── ens/          # 集成学习框架
└── interpret/    # 模型可解释性（SHAP 等）
    └── base.py   # LightGBMFInt（特征重要性接口）
```

所有模型实现类（LightGBM、LSTM 等）都继承自 `base.py` 中的 `Model`，必须实现 `fit(dataset)` 和 `predict(dataset)` 方法。

### 4.5 `qlib/backtest/` — 回测引擎

```
qlib/backtest/
├── backtest.py        # backtest_loop(), collect_data_loop() 主循环
├── account.py         # Account（资金账户状态管理）
├── position.py        # Position（持仓管理）
├── exchange.py        # Exchange（模拟交易所：撮合、成本计算）
├── executor.py        # BaseExecutor, SimulatorExecutor（模拟执行器）
├── decision.py        # Order, TradeDecision（交易决策数据结构）
├── signal.py          # Signal（信号包装）
├── report.py          # 回测报告生成
├── profit_attribution.py  # 收益归因
├── high_performance_ds.py # 高性能数据结构（优化回测速度）
└── utils.py           # 工具函数
```

**回测执行模式**：Qlib 支持 **Nested Decision Execution（嵌套决策执行）**，即在外层日频策略中嵌套内层分钟频执行器，支持高频仿真。

**核心执行流程**：
```python
# 典型的回测调用链
backtest_loop(start_time, end_time, trade_strategy, trade_executor)
  → trade_strategy.generate_trade_decision(execute_result)
  → trade_executor.execute(trade_decision)
  → Exchange.deal_order(order) → 更新 Account/Position
```

### 4.7 `qlib/workflow/` — 工作流与实验记录

```
qlib/workflow/
├── __init__.py   # 导出 R（Recorder 全局入口）
├── exp.py        # Experiment（实验抽象）
├── expm.py       # ExperimentManager（实验管理器）
├── recorder.py   # Recorder（记录器，封装 MLflow）
├── record_temp.py # 预置 Record 模板：SignalRecord, SigAnaRecord, PortAnaRecord
├── task/         # 任务管理（超参搜索、分布式任务等）
└── utils.py      # 工具函数
```

**Recorder（R）** 是 Qlib 的实验管理模块，基于 MLflow 实现：
- `R.start(experiment_name)` → 开启实验
- `R.save_objects(model=model)` → 保存模型
- `R.log_metrics(IC=0.05)` → 记录指标
- `R.end()` → 结束实验

## 8. 数据存储格式：`.bin` 文件

Qlib 设计了专有的二进制数据格式（`.bin`），相比通用数据库（MySQL、CSV）有显著性能优势。

**基准测试**（构建 14 因子数据集，800 支股票，2007–2020）：

| 方案 | 耗时（1 CPU） | 耗时（64 CPU） |
|------|-------------|--------------|
| MySQL | 184 秒 | — |
| HDF5 | 365 秒 | — |
| MongoDB | 253 秒 | — |
| Qlib（+缓存） | **7.4 秒** | — |

**文件组织结构**：
```
features/
└── sh600000/
    ├── open.day.bin     # 开盘价序列（复权）
    ├── close.day.bin    # 收盘价序列（复权）
    ├── high.day.bin
    ├── low.day.bin
    ├── volume.day.bin
    └── factor.day.bin   # 复权因子（= 复权价 / 原始价）
```

每个 `.bin` 文件存储一个股票的单个字段的所有历史数据，按时间顺序对齐到交易日历（`calendars/day.txt`）。

## 9. 核心工作流：端到端使用流程

### 9.1 自动工作流（推荐新手）

使用 `qrun` 命令，指定 YAML 配置文件，一键完成**数据处理→模型训练→回测→评估**：

```bash
cd examples
qrun benchmarks/LightGBM/workflow_config_lightgbm_Alpha158.yaml
```

YAML 配置文件结构如下（以 LightGBM Alpha158 为例）：

```yaml
qlib_init:                     # Qlib 初始化参数
  provider_uri: ~/.qlib/qlib_data/cn_data
  region: cn
market: &market csi300         # 股票池（沪深300）
benchmark: &benchmark SH000300 # 基准指数
data_handler_config: ...       # DataHandler 配置（时间范围、特征等）
port_analysis_config: ...      # 回测配置（策略、资金、成本等）
task:
  model: ...                   # 模型类及超参
  dataset: ...                 # 数据集类（DatasetH + Alpha158）
  record: ...                  # 记录模板（SignalRecord, SigAnaRecord, PortAnaRecord）
```

### 9.2 代码工作流（灵活定制）

通过 Python 代码构建工作流，适合深度定制：

```python
import qlib
from qlib.constant import REG_CN
from qlib.contrib.model.gbdt import LGBModel
from qlib.contrib.data.handler import Alpha158
from qlib.data.dataset import DatasetH
from qlib.workflow import R

# 1. 初始化
qlib.init(provider_uri="~/.qlib/qlib_data/cn_data", region=REG_CN)

# 2. 构建 DataHandler
handler = Alpha158(
    instruments="csi300",
    start_time="2008-01-01",
    end_time="2020-08-01",
    fit_start_time="2008-01-01",
    fit_end_time="2014-12-31",
)

# 3. 构建 Dataset（train/valid/test 分割）
dataset = DatasetH(
    handler=handler,
    segments={
        "train": ("2008-01-01", "2014-12-31"),
        "valid": ("2015-01-01", "2016-12-31"),
        "test":  ("2017-01-01", "2020-08-01"),
    },
)

# 4. 模型训练
model = LGBModel(loss="mse", num_leaves=210, learning_rate=0.2)
with R.start(experiment_name="lightgbm_alpha158"):
    model.fit(dataset)
    R.save_objects(model=model)

# 5. 预测与评估
pred = model.predict(dataset)
```

## 10. 两种运行模式

| 模式 | 描述 | 适用场景 |
|------|------|---------|
| **Offline 模式**（默认） | 数据部署在本地，离线批量处理 | 研究/回测 |
| **Online 模式** | 数据部署为共享服务（Qlib-Server），多客户端共享缓存 | 生产环境/多人协作 |

在线模式需要额外部署 [Qlib-Server](https://github.com/microsoft/qlib-server)，支持 Azure 一键部署。

## 11. 强化学习框架（RL）

Qlib 的 RL 框架专注于**订单执行优化**场景：

- **场景**：给定一个大订单（如卖出 10 万股），如何在一个交易日内分批执行以最小化冲击成本？
- **内置方法**：TWAP（时间加权均价）、PPO（近端策略优化）、OPDS（Oracle Policy Distillation）
- **环境**：利用 Qlib 回测引擎提供分钟级市场仿真环境
- **嵌套执行**：RL 执行器可嵌套在日频策略下，外层做组合选股，内层 RL 做订单执行

---

## 12. 元学习框架（Market Dynamics Modeling）

针对金融市场**非平稳性**（分布漂移）问题，Qlib 提供：

- **Rolling Retraining**（滚动再训练）：定期用最新数据重新训练模型
- **DDG-DA**（Dynamic Dataset Generation with Distribution Alignment, AAAI 2022）：自动生成适配当前市场分布的训练数据

## 13. 扩展与贡献

Qlib 设计为高度可扩展：

| 扩展点 | 方式 |
|--------|------|
| 新增模型 | 继承 `qlib.model.base.Model`，实现 `fit/predict` |
| 新增策略 | 继承 `qlib.strategy.base.BaseStrategy`，实现 `generate_trade_decision` |
| 新增处理器 | 继承 `qlib.data.dataset.processor.Processor`，实现 `fit/transform` |
| 新增数据源 | 编写爬虫脚本，使用 `dump_bin.py` 转换格式 |
| 新增因子 | 在 YAML/代码中直接用表达式引擎语法定义 |
